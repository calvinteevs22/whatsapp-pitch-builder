PROBLEM:
When the image proxy returns a response that is NOT an image error (e.g., returns 
a 1x1 transparent PNG, or a redirect page), the <img> tag doesn't fire onError 
because it technically "loaded" something. The result is:
- If proxy returns 1x1 transparent PNG → shows tiny transparent pixel on gray bg
- If proxy returns HTML error page → may show broken image or green bg
- If the crawled URL itself returns a non-image (e.g., HTML redirect) → similar issue

But the user's screenshot shows a SOLID BRIGHT GREEN block, not the emerald-50 gradient.
This suggests the image proxy might be returning something that renders as green.
